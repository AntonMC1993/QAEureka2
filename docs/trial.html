Formato de Pruebas de Software



A continuación, mostraremos un ejemplo de un formato de Prueba Manual realizado en TestLink para una API REST desarrollada en SpringBoot por Nicole Opazo; Desarrolladora Full Stack. Como se trata de una aplicación sencilla, de fácil entendimiento, con entradas y salidas sin restricciones; los errores son muy pocos. Por ejemplo, al registrar un nombre de usuario compuesto sólo con números, la respuesta es positiva y no presenta errores de registro. Sin embargo, al intentar registrar en el campo "edad" una respuesta compuesta por letras, se presenta un error y no permite realizar el registro de usuario nuevo.
