¿Qué tipos de pruebas es posible aplicar durante el proceso de desarrollo y ejecución de un software?



Durante el proceso de desarrollo y ejecución de un software, se pueden aplicar varios tipos de pruebas para asegurarse de que el software funcione correctamente y cumpla con los requisitos. A continuación, se presentan algunas clasificaciones de los tipos de pruebas más comunes:

    Pruebas Unitarias (Unit Testing):
  • Propósito: Verificar que las unidades individuales de código, como funciones o métodos, funcionen correctamente
  • Alcance: Prueba de componentes aislados del software.
  • Herramientas: Frameworks de pruebas unitarias como JUnit, NUnit, pytest, etc.
    • Pruebas de Integración (Integration Testing):
  • Propósito: Comprobar que los diferentes componentes o módulos del software se integren adecuadamente y funcionen juntos.
  • Alcance: Prueba la interacción entre componentes.
  • Herramientas: Pueden utilizarse herramientas de automatización de pruebas y marcos de pruebas.
    • Pruebas Funcionales (Functional Testing):
  • Propósito: Evaluar si el software cumple con los requisitos funcionales especificados.
  • Alcance: Enfoque en la funcionalidad desde la perspectiva del usuario final.
  • Herramientas: Pruebas manuales o herramientas de automatización de pruebas como Selenium, Cypress, etc.
    • Pruebas de Aceptación (Acceptance Testing):
  • Propósito: Confirmar que el software cumple con los criterios de aceptación y las expectativas del cliente.
  • Alcance: Puede incluir pruebas de aceptación del usuario (UAT) realizadas por usuarios finales.
  • Herramientas: A menudo son pruebas manuales realizadas por el cliente o equipos de pruebas dedicados.
    • Pruebas de Regresión (Regression Testing):
  • Propósito: Asegurarse de que las modificaciones en el código no introduzcan regresiones o errores nuevos en áreas previamente funcionales.
  • Alcance: Reprueba funcionalidades existentes.
  • Herramientas: Pueden utilizarse herramientas de automatización para acelerar las pruebas de regresión.
    • Pruebas de Rendimiento (Performance Testing):
  • Propósito: Evaluar el rendimiento, la velocidad y la escalabilidad del software.
  • Alcance: Prueba el rendimiento bajo diferentes cargas y escenarios.
  • Herramientas: Herramientas de pruebas de carga y rendimiento como Apache JMeter, LoadRunner, etc.
    • Pruebas de Seguridad (Security Testing):
  • Propósito: Identificar vulnerabilidades y asegurar que el software sea resistente a ataques y cumpla con estándares de seguridad.
  • Alcance: Se enfoca en la seguridad de datos, autenticación, autorización, etc.
  • Herramientas: Herramientas de pruebas de seguridad como OWASP ZAP, Nessus, etc.
    • Pruebas de Usabilidad (Usability Testing):
  • Propósito: Evaluar la facilidad de uso, la eficiencia y la satisfacción del usuario al interactuar con el software.
  • Alcance: Implica a usuarios reales que realizan tareas típicas del sistema.
  • Herramientas: Pruebas conducidas por observadores y herramientas de registro de interacciones.
Estos son algunos de los tipos de pruebas que se pueden aplicar en el proceso de desarrollo y ejecución de software. La elección de qué tipos de pruebas aplicar depende de los objetivos del proyecto, los requisitos del cliente y las características específicas del software. En muchos casos, se utilizan múltiples tipos de pruebas en combinación para garantizar la calidad y el rendimiento del software.

Sin embargo, si nos detenemos en otra clasificación más particular, podemos señalar que dentro de los tipos de testing, encontramos los del tipo Manual y Automatizado.

  • Testing Manual:
    • Es el proceso de probar software a mano, para obtener más información al respecto y descubrir qué funciona y qué no. Por lo general esto incluye la verificación de todas las características especificadas en los documentos de requisitos. Además los testers prueban el software con la perspectiva de usuario final. Muchos tienen guiones completos, guías que señalan los resultados esperados.
  • Testing Automatizado:
    • Es el proceso de probar el software con una herramienta de automatización para encontrar defectos. En este proceso se ejecutan scripts de prueba y generarán los resultados de forma automática. Algunas de las herramientas más famosas para pruebas funcionales son: Cutty, UTP y Selenium.

    Métodos de Testing:
  • Testing Estático: Se conocen también como verificación en pruebas de software. Sirven para verificar documentos y archivos. Es el proceso para asegurarnos de que estamos desarrollando el proyecto correctamente, es decir, para verificar los requisitos que tenemos y verificar si estamos desarrollando el producto en consecuencia o no. Estas prácticas incluyen inspecciones, revisiones, tutoriales y demáses.
  • Testing Dinámico: Se conocen también como validación de pruebas de software. Es un proceso de validación del producto real, es el proceso por el cual nos aseguramos de estar construyendo el producto correcto. Y las actividades involucradas son probar la aplicación de software, no sólo vemos secciones por si mismas, sino que testeamos en conjunto todas las funcionalidades.

    • Prácticas de Testing:

  • Pruebas de caja blanca: Se basa en la estructura del código interno de la aplicación. Se utiliza perspectiva interna del sistema, así como habilidades de programación para diseñar casos de prueba. Esta prueba se realiza a nivel de unidad, también conocido como Unit Testing. Acá se evalúa la entrada y salida verificando lo que sucede internamente.

  • Pruebas de caja negra: También denominadas como pruebas de comportamiento, están basadas en especificaciones. También conocidas como pruebas de entrada y salida. Se evalúa la funcionalidad del software sin mirar la estructura del código interno. Acá se evalúa la entrada y salida sin mirar lo que suceda de forma interna.

  • Pruebas de caja gris: Es una combinación de las dos anteriores, se tiene acceso a la documentación del diseño y al código. Esto nos ayuda a crear mejores casos de prueba durante el proceso de testing.
